# План разработки административной панели TravelLine

## Цель

Разработать простую административную панель для управления содержимым сайта.

При разработке придерживаемся следующих принципов:

* минимум JavaScript;
* серверный рендеринг (FastAPI + Jinja2);
* каждая страница отвечает только за одну задачу;
* простая архитектура без SPA и React/Vue;
* использование уже разработанного REST API.

---

# Общая структура

Административная панель состоит из трех уровней.

```
Список секций
        ↓
Список элементов секции
        ↓
Редактирование элемента
```

Навигация осуществляется обычными гиперссылками.

```
/admin
/admin/{section}
/admin/{section}/{element_id}
```

---

# Этап 1. Страница списка секций +

## URL

```
GET /admin
```

## Возможности

* отображение всех существующих секций;
* переход к выбранной секции;
* минималистичный интерфейс.

Пример:

```
Hero        >

About       >

Services    >

Team        >

Footer      >
```

---

# Этап 2. Страница элементов секции

## URL

```
GET /admin/{section}
```

## Возможности

Отображение всех элементов выбранной секции.

Для каждого элемента выводится:

* тип;
* краткое содержимое;
* позиция;
* кнопка редактирования.

Также отображается кнопка

```
+ Добавить элемент
```

---

# Этап 3. Создание элемента

## Интерфейс

На странице списка элементов секции располагается кнопка

```text
+ Добавить элемент
```

После нажатия открывается форма создания нового элемента.

## Возможности

Пользователь может заполнить доступные поля элемента:

* тип;
* позицию;
* заголовок;
* текст;
* изображение;
* ссылку.

После отправки формы вызывается

```text
POST /api/sections/{section_name}/elements
```

При успешном создании пользователь возвращается к списку элементов выбранной секции.

---

# Этап 4. Просмотр и редактирование элемента

## URL

```text
GET /admin/{section}/{element_id}
```

При открытии страницы сначала загружается элемент через

```text
GET /api/sections/{section_name}/elements/{element_id}
```

После изменения данных вызывается

```text
PATCH /api/sections/{section_name}/elements/{element_id}
```

Поддерживается изменение любого из следующих полей:

* type;
* position;
* heading;
* text;
* image;
* link.

После успешного сохранения пользователь остается на странице редактирования элемента.

---

# Этап 5. Удаление элемента

На странице редактирования располагается кнопка

```text
Удалить
```

После подтверждения удаления вызывается

```text
DELETE /api/sections/{section_name}/elements/{element_id}
```

При успешном удалении пользователь автоматически возвращается к списку элементов текущей секции.


# Этап 6. Управление порядком элементов

Каждый элемент имеет поле

```
position
```

Изменение позиции реализуется обменом двух элементов местами.

Пример

До

```
1
2
3
```

После переноса третьего элемента на первое место

```
3
2
1
```

Никакого полного пересчета позиций не производится.

---

# Этап 7. Ограничение доступных полей

Каждая секция имеет собственный набор разрешенных полей.

Например

```
hero

text
```

```
team

heading
text
image
```

Форма редактирования строится автоматически на основании конфигурации секции.

---

# Этап 8. Работа с изображениями

Изображения загружаются через Drag & Drop.

Поддерживается:

* PNG;
* JPEG.

Алгоритм:

```
Файл

↓

FileReader

↓

Base64

↓

POST/PATCH

↓

База данных
```

На первом этапе изображения сохраняются в текстовом поле БД в формате Base64.

---

# Этап 9. Интерфейс

Минимальный набор действий:

```
Редактировать

Удалить

Добавить элемент
```

При необходимости позднее могут быть добавлены:

* кнопки "Вверх";
* кнопки "Вниз";
* предпросмотр изображений;
* поиск по секциям.
